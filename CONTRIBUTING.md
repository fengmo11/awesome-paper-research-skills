# Contributing

Thanks for helping improve this list.

## Add A Project

Open an issue or pull request with:

- Project name and GitHub URL.
- One-sentence description.
- Workflow stages covered.
- Observed stars/forks and date checked.
- Evidence for any claims about citation verification, experiments, LaTeX,
  DOCX, or peer review.

## Preferred Categories

- Full-cycle AI scientist systems.
- Paper writing and drafting.
- Literature search and citation infrastructure.
- Review and quality tools.
- Research radar and monitoring.
- Templates, datasets, and benchmarks.

## Quality Bar

Projects should be open source and useful for at least one real research
workflow stage. Small projects are welcome if they solve a clear problem.

## Style

- Keep descriptions factual.
- Avoid hype-only language.
- Mention human review requirements for autonomous systems.
- Prefer verified links over second-hand summaries.
