# Selection Guide

## If You Want To Build A Public GitHub Project

Choose a narrow, verifiable workflow. The best GitHub projects in this space are
not just "AI writes paper"; they give users traceability, citations, exports, or
review artifacts.

Good first projects:

- `paper-source-map`: PDF or topic to claim-evidence-citation matrix.
- `arxiv-field-radar`: scheduled paper tracker with summaries and trends.
- `bibtex-ref-auditor`: BibTeX and reference quality checker.
- `paper-review-scorecard`: PDF to structured reviewer report.
- `latex-paper-packager`: Markdown and BibTeX to venue-ready LaTeX package.

## If You Need Literature Search

Start with:

- paper-search-mcp
- litdb
- OpenAlex
- Semantic Scholar
- arXiv

Prioritize source metadata and DOI validation before generation.

## If You Need Writing

Start with:

- OpenDraft for long-form draft/export patterns.
- hermes-agent for skill design and ML/AI venue writing patterns.
- RE-paper-writing for modular claim-evidence and submission QA patterns.

## If You Need Experiments

Start with:

- AI-Scientist for the canonical autonomous loop.
- AutoResearchClaw for an idea-to-paper architecture.
- data-to-paper for traceability from raw data to claims.

## If You Need Review

Start with:

- ai-peer-review for review generation.
- A custom reviewer panel with methodology, domain, statistics, novelty, and
  writing reviewers.
- A final evidence audit that checks every strong claim against a source or
  experiment result.
